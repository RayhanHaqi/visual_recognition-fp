\section{Introduction}

This report documents our \textbf{NOAA Fisheries Steller Sea Lion Population Count} solution (Final Project, Topic~3), Visual Recognition using Deep Learning (2026 Spring). The task is to predict five non-negative integer counts per aerial photograph:

\begin{itemize}
    \item \texttt{adult\_males}, \texttt{subadult\_males}, \texttt{adult\_females}, \texttt{juveniles}, \texttt{pups}
\end{itemize}

Performance is evaluated by \textbf{Root Mean Squared Error (RMSE)} on a hidden test set; \textbf{lower is better}. The competition had 386 teams; the historical \textbf{golden medal} band on the public leaderboard is RMSE $\approx$ 15.88--10.86 (best $\approx$ 10.86).

\subsection{Course Grading Context (Topic 3, Detection)}
For our presentation slot (2026/06/02), course grading allocates 50\% to model performance: golden medal $\Rightarrow$ 25 points, top-3 $\Rightarrow$ 40 points, and within \textbf{3\%} of the best score on the leaderboard $\Rightarrow$ 50 points (detection competitions use the 3\% band). Presentation (30\%), report \& code (10\%), and peer review (10\%) follow the rubric in \texttt{(114-2)FinalProject.pdf} and \texttt{Tips\_FinalProj\_Presentation.pdf}: required sections are Introduction, Related Work, Method, Experimental Results (with Kaggle rank screenshot), Conclusion, References, code link, and team contribution table.

\subsection{Task and Dataset}
The dataset contains high-resolution aerial images of Steller sea lion colonies:
\begin{itemize}
    \item \textbf{Training:} 949 original images with dot annotations in \texttt{TrainDotted/}; 114 mismatched pairs are removed using \texttt{MismatchedTrainImages.txt}, leaving \textbf{891} usable training images.
    \item \textbf{Test:} 18,636 unlabeled images in \texttt{Test/}.
    \item \textbf{Size:} approximately 103~GB compressed; extracted images occupy $\sim$96~GB on disk.
    \item \textbf{Resolution:} variable across images (multi-megapixel aerial frames); images are much larger than typical ImageNet inputs.
\end{itemize}

Each training image is paired with five class counts in \texttt{train.csv}. Unlike bounding-box detection, the model must regress population counts directly from pixels.

\subsection{Motivation and Difficulties}
\textbf{Importance.} Automated counts from aerial surveys support fisheries monitoring of the endangered Steller sea lion; manual dot counting on megapixel mosaics is slow and error-prone.

\textbf{Challenges we address.}
\begin{itemize}
    \item \textbf{Extreme image size:} a single frame can exceed 4K resolution; naive whole-image resize destroys small animals.
    \item \textbf{Weak default supervision:} scaling image-level counts to random crops (area fraction) does not use dot locations in \texttt{TrainDotted/}.
    \item \textbf{Train/test distribution shift:} test-set images are downscaled in strong solutions; inference must tile consistently with training.
    \item \textbf{Metric mismatch:} tile-level training RMSE can look excellent ($\sim$0.2) while image-level validation and Kaggle RMSE remain high ($\sim$30+) unless aggregation and labels are aligned.
\end{itemize}

\section{Related Work}

\subsection{Tile-Based Counting Regression}
Top Kaggle solutions treat sea lion counting as \textbf{density / tile regression} rather than end-to-end image classification. The first-place approach (outrunner) and subsequent strong baselines train CNNs on fixed-size image tiles and aggregate tile predictions to obtain image-level counts \cite{outrunner2017}.

\subsection{UNet + Secondary Regressor (2nd Place)}
Lopuhin \cite{lopuhin2017} proposed a two-stage pipeline: a UNet predicts segmentation-style density maps on scaled training images, then a secondary regressor maps aggregated UNet outputs to the five count targets. This approach achieved second place and remains a strong reference for handling large images via downscaling and dense prediction.

\subsection{Inception-ResNet v2 Tiles + Ensemble (4th Place)}
Asanakoy \cite{asanakoy2017} fine-tuned Inception-ResNet v2 on $299\times299$ tiles with RMSE loss, using heavy augmentation and multi-shift test-time tiling. Test images were downscaled to $\sim$0.4--0.5$\times$ before inference. An ensemble of models trained with different scale augmentations improved robustness.

\subsection{Design Implications for Our Pipeline}
These works share several principles adopted in our implementation:
\begin{enumerate}
    \item \textbf{Never resize the full image to a tiny square} --- use tiling or downscaling instead.
    \item \textbf{Train on tiles}, predict on overlapping or shifted tile grids at test time.
    \item \textbf{Downscale test images} for faster inference and better generalization (we use $0.5\times$ test scaling in production runs).
    \item \textbf{Multi-shift TTA} at inference improves RMSE by averaging predictions from staggered tile grids.
\end{enumerate}

\section{Method}

\subsection{Overview}
We use a \textbf{tile regression} pipeline: a timm CNN backbone (ResNet-50 for early baselines; \textbf{Inception-ResNet v2} for the best run) with a 5-dimensional count head, trained on $299\times299$ crops and evaluated by tiled inference with multi-shift TTA. The head is either a single linear layer or an Asanakoy-style MLP (\texttt{Dropout} $\rightarrow$ 256 $\rightarrow$ \texttt{ReLU} $\rightarrow$ \texttt{Dropout} $\rightarrow$ 5) when \texttt{--head\_hidden 256} is set.

\subsection{Data Preprocessing}
\begin{enumerate}
    \item Download \texttt{KaggleNOAASeaLions.7z} ($\sim$96~GB) via authenticated streaming (\texttt{curl}), not the Kaggle Python client (which buffers the full file in RAM).
    \item Extract password-protected archive into \texttt{datasets/}.
    \item Fetch corrected \texttt{train.csv} labels (official API file unavailable in 2026); map legacy \texttt{subadult\_females} $\rightarrow$ \texttt{juveniles} where needed.
    \item Remove mismatched training images listed in \texttt{MismatchedTrainImages.txt}.
\end{enumerate}

\subsection{Training: Random Tile Regression}
We use \texttt{SeaLionTileDataset} with \texttt{--use\_tiles}:
\begin{itemize}
    \item Load full-resolution train image.
    \item Sample a random $299\times299$ crop (native pixels, not whole-image resize).
    \item Target = full image counts $\times$ (crop area / image area).
    \item Augmentations: random horizontal/vertical flip, 90$^\circ$ rotations, \texttt{ColorJitter}.
    \item Eight random tiles per image per epoch ($757 \times 8 \approx 6056$ steps with 85/15 train/val split).
\end{itemize}

This teaches the network to predict \emph{how many sea lions appear in a local patch}, consistent with top competition solutions.

\subsection{Dot Labels and Balanced Tile Training (Phases 1--3)}
Area-fraction targets (v1/v2) scale image-level counts by crop size but do not use pixel-level dot locations. Top solutions supervise tiles with dot counts inside each crop.

\textbf{Dot cache (Phase 1).} Hand-tuned HSV extraction from \texttt{Train}/\texttt{TrainDotted} pairs was unstable. We instead import the community coordinate file \texttt{coords-threeplusone-v0.4.csv} from Lopuhin's 2nd-place repository \cite{lopuhin2017} via \texttt{scripts/import\_dot\_coords.py}, producing \texttt{datasets/dot\_labels.csv} ($\sim$81k dots, 891 images). A class-level gate (\texttt{analyze\_dot\_cache.py --fail\_on\_gate}) verifies agreement with \texttt{train.csv} (all classes relative error $<1\%$).

\textbf{Balanced dots (Phase 2, best).} \texttt{label\_mode=balanced\_dots} samples dot-centered positive crops and background crops with zero targets, reducing dominance of empty tiles. Training uses ResNet-50, 20 epochs, batch 256, $lr=10^{-4}$, same tiled validation and 5-shift inference as v2.

\textbf{Gaussian soft dots (Phase 3).} \texttt{label\_mode=gaussian\_dots} replaces hard dot counts in a crop with Gaussian-weighted mass by distance to the crop window. This did not improve Kaggle RMSE (Section~\ref{sec:results}).

\subsection{Stronger Backbones (Phases 4 and 6)}
After dot supervision (v5), we swapped the backbone while keeping \texttt{balanced\_dots}, scaled test inference, 5-shift TTA, and pup$\times$1.2:
\begin{itemize}
    \item \textbf{Phase 4:} \texttt{tf\_efficientnet\_b3\_ns}, 20 epochs, batch 128 $\Rightarrow$ private RMSE \textbf{15.16}.
    \item \textbf{Phase 6:} \texttt{inception\_resnet\_v2} (Asanakoy architecture), 20 epochs, batch 128 $\Rightarrow$ private RMSE \textbf{14.58} (best).
\end{itemize}
Backbone capacity helped once labels were fixed; ResNet-50 with the same dot pipeline (v5) remained at 17.41.

\subsection{Top-3 Push (Phases 7--7b, In Progress)}
To approach the course top-3 band (private RMSE $<13.05$), we added:
\begin{itemize}
    \item \textbf{256-hidden regression head} and dropout 0.5 (checkpoint-compatible via \texttt{build\_counter\_from\_checkpoint\_args}).
    \item \textbf{Source-crop scale augmentation:} sample a larger/smaller window, count dots in that window, resize to $299\times299$ (\texttt{--scale\_min}/\texttt{--scale\_max}, Asanakoy uses 0.83--1.25 and 0.66--1.5 variants).
    \item Lab scripts \texttt{run\_phase7\_inception\_v9.sh}, \texttt{run\_phase7\_inception\_v10.sh}, and \texttt{run\_top3\_train\_queue.sh}; quick inference sweeps (\texttt{run\_top3\_quick\_sweeps.sh}) on \texttt{Test\_scaled\_0.4} without retraining.
\end{itemize}
Results for v9/v10 are pending at report writing time.

\subsection{Validation and Inference: Tiled Aggregation}
For full-image evaluation, we slide a $299\times299$ window across each image. Each tile is forwarded through the network; tile outputs are combined into one 5-D count vector.

\textbf{Initial implementation issue:} summing predictions from \textbf{overlapping} tiles (stride 149, multiple shifts) while training on non-overlapping random crops caused validation RMSE to \textbf{increase} as training improved (tile predictions grew, overlap over-counted). Table~\ref{tab:broken_val} shows this failure mode on run \texttt{fp\_resnet50\_e30\_bs128\_t299}.

\textbf{Fix (current codebase):}
\begin{itemize}
    \item \textbf{Training validation:} \texttt{val\_shifts=1}, \texttt{val\_stride=tile\_size} (non-overlapping grid) for a meaningful RMSE monitor.
    \item \textbf{Inference TTA:} sum tiles \emph{within each shift}, then \textbf{average} shift-level image counts (not one global sum over all shifts).
    \item \textbf{Final inference:} \texttt{--shifts 5 --stride 299}.
\end{itemize}

\subsection{Loss and Optimization}
\begin{itemize}
    \item \textbf{Loss:} RMSE between predicted and target 5-D count vectors.
    \item \textbf{Optimizer:} AdamW ($lr = 10^{-4}$, weight decay $10^{-4}$).
    \item \textbf{Schedule:} OneCycleLR over all training steps ($pct\_start = 0.1$).
    \item \textbf{Mixed precision:} AMP enabled on GPU.
    \item \textbf{DataLoader:} \texttt{workers=0} on CUDA to avoid fork-related deadlocks after epoch~1.
\end{itemize}

\subsection{Test-Time Pipeline and Inference Engineering}
\textbf{Production inference (best: inception\_v8).} Test images are downscaled to $0.5\times$ (\texttt{Test\_scaled\_0.5/}) \cite{asanakoy2017}. We tile with $299\times299$ windows, stride 299, \textbf{5-shift TTA}: sum tile predictions within each shift, average across shifts, then sum unique tiles to image counts (\texttt{data/predict.py}). Optional pup calibration ($\times 1.2$) is applied via \texttt{data/calibrate\_submission.py}. Phase~5 CSV ensembling with weaker checkpoints (v4, v6) increased RMSE; we submit the single best strong model.

Full test inference covers 18{,}636 images ($\sim$600+ tiles per image with 5 shifts). Profiling on 200 images (batch 512, scaled test) shows $\sim$62\% time in CPU tile preprocessing (resize/normalize), $\sim$24\% GPU forward, $\sim$1.3--1.4 images/s ($\sim$4~h total). We added \texttt{cv2} resize, optional \texttt{--amp}, and pinned-memory host-to-device copies to reduce overhead.

\subsection{Hardware and Reproducibility}
Training runs on lab machine \textbf{w61} (NVIDIA RTX 5090; runs use \texttt{cuda:1}). Code, logs (\texttt{log/*.csv}), and scripts are in Git; datasets, checkpoints, and \texttt{.7z} archives remain local (gitignored). Kaggle credentials are stored in a gitignored workspace directory (\texttt{FP/.kaggle/}).

\subsection{Ablation Studies and Negative Results}
\begin{itemize}
    \item \textbf{Test downscaling (v4):} Using the v2 checkpoint on \texttt{Test\_scaled\_0.5} improved private RMSE from 28.09 to 25.54 without retraining.
    \item \textbf{Pup calibration:} Scaling pup predictions by 1.2 helped v4/v5 on the leaderboard ($\approx$0.1--0.2 RMSE gain on v5).
    \item \textbf{Ensembling with v4:} Blending v5 with the weaker v4 submission monotonically increased RMSE (best blend 17.61 vs.\ 17.41 pure v5).
    \item \textbf{Gaussian dot targets (v6):} Softer supervision hurt both validation and Kaggle scores vs.\ hard balanced dots.
    \item \textbf{Backbone swap (Phases 4/6):} With fixed dot labels, Inception-ResNet v2 beat ResNet-50 by $\sim$2.8 RMSE private (17.41 $\rightarrow$ 14.58).
    \item \textbf{Validation metric:} Image-level val RMSE on 134 holdout images is a poor proxy for 18k-image Kaggle RMSE once dot-based training is used.
\end{itemize}

\section{Conclusion}

We built a reproducible tile-regression pipeline for the NOAA Steller Sea Lion Population Count competition, following top Kaggle solutions: large-image tiling, test downscaling, multi-shift TTA, and dot-aware tile supervision. Engineering fixes (streaming download, submission schema, non-overlapping validation, shift-averaged TTA) were necessary before model improvements appeared on the leaderboard.

Our best result uses community dot coordinates \cite{lopuhin2017}, balanced dot-centered tile training, \textbf{Inception-ResNet v2} inference on $0.5\times$ scaled test images, and pup count calibration, achieving Kaggle private RMSE \textbf{14.58} (public 14.47)---a $\sim$48\% relative improvement over the area-label baseline (28.09) and $\sim$16\% over ResNet-50 with the same labels (17.41). EfficientNet-B3 reached 15.16. Test-time ensembling with weaker models and Gaussian soft-dot targets did not help; backbone choice after fixing supervision was the largest remaining gain.

\textbf{Findings:} (1)~Dot-aligned tile targets matter more than backbone or area-fraction scaling; (2)~Inception-ResNet v2 matches the public 4th-place recipe better than ResNet-50; (3)~test downscaling and pup calibration are cheap inference-time gains; (4)~ensembling requires comparably strong checkpoints; (5)~CPU tile preprocessing dominates inference ($\sim$4~h for 18k images). Phase~7 (hidden head + scale augmentation) targets the course top-3 band ($<13.05$ private RMSE). We include the required code link, Kaggle snapshot, contribution table, and ablations above.
